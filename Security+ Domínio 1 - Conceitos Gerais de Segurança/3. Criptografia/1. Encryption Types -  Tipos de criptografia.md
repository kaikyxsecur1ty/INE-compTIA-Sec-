[[1. Criptografia de chave simétrica]]

___

1. **Fundamentos e Propósitos**
	A criptografia é o estudo dos métodos para codificar e decodificar dados

	**Conceitos Fundamentais:**
	• **Plaintext:** Dados originais legíveis
	• **Ciphertext:** Dados criptografados e ilegíveis
	• **Key (Chave):** Dados usados pelo algoritmo para codificar e decodificar
	
	**Objetivos de Segurança:** A criptografia atende a vários princípios de segurança
	1. **Confidencialidade:** O texto criptografado não é legível sem a chave de descriptografia
	2. **Integridade:** Verificação de hash garante que os dados não foram alterados
	3. **Autenticidade:** Pares de chaves pública/privada verificam a origem dos dados
	4. **Não-repúdio:** Chaves e assinaturas provam que um indivíduo ou sistema executou uma ação
	5. **Autorização:** Certificados podem ser usados para comprovar identidade e conceder acesso

A proteção deve ser aplicada aos dados em três estados: em repouso (armazenamento), em trânsito (rede) e em uso (memória RAM).

2. **Tipos de Criptografia**

	O Security+ exige a compreensão da diferença entre os dois principais tipos:

| **Característica** | **Simétrica**                                                   | **Assimétrica**                                                               |
| :----------------: | :-------------------------------------------------------------- | :---------------------------------------------------------------------------- |
|     **Chaves**     | Usa a mesma chave para criptografar e descriptografar.          | Usa chaves diferentes: uma **pública (compartilhada e uma privada (secreta)** |
|   **Velocidade**   | Mais rápida                                                     | Mais lenta                                                                    |
|    **Funções**     | Oferece confidencialidade, mas não autenticidade ou integridade | Oferece confidencialidade, autenticidade e não-repúdio                        |
|    **Exemplos**    | AES, DES, 3DES                                                  | Diffie-Hellman, RSA                                                           |

O **PKI (Public Key Infrastructure)** usa a criptografia assimétrica para proteger a troca de chaves que serão usadas na criptografia simétrica (modelo híbrido)

3. **Infraestrutura de Chave Pública (PKI)**
	PKI é uma infraestrutura que visa estabelecer e manter um alto nível de confiança nas chaves públicas
	
	• **CA (Certificate Authority):** Uma entidade confiável que emite _Certificados Digitais_ (geralmente X.509)
	• **Certificado Digital:** Associa uma chave pública a um proprietário
	• **CSR (Certificate Signing Request):** Um pedido enviado a uma CA, contendo a chave pública e informações adicionais (como nome comum e organização)
	• **Revogação:** Certificados comprometidos ou substituídos são listados em uma **CRL (Certificate Revocation List)** ou verificados via **OCSP (Online Certificate Status Protocol)**

3. **Ferramentas e Práticas**
	Existem ferramentas e métodos importantes relacionados à criptografia:
	
	• **Hashing:** Uma função unidirecional usada para verificar a integridade de arquivos ou mensagens. Não pode ser revertida
	• **Salting:** Adicionar caracteres aleatórios a uma senha antes de aplicar o hash para dificultar ataques de força bruta usando tabelas pré-calculadas
	• **Digital Signatures (Assinaturas Digitais):** Usam criptografia assimétrica para verificar autenticidade e integridade. O autor usa sua chave privada para criptografar o hash do item
	• **TPM (Trusted Platform Module):** Um chip integrado que fornece armazenamento seguro para chaves de criptografia e pode autenticar a plataforma de hardware
	• **HSM (Hardware Security Module):** Um dispositivo de hardware físico separado e reforçado, dedicado ao gerenciamento de chaves e operações criptográficas
	• **Blockchain:** Um registro de transações armazenadas criptograficamente, onde cada bloco está ligado ao anterior por meio de um hash, garantindo a integridade e transparência
