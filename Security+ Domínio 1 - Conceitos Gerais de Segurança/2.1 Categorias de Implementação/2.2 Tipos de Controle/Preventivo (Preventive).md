1. **Definição e Função do Controle Preventivo**
A função principal de um **Controle Preventivo** é operar **antes que um ataque possa ocorrer**

- Eles são projetados para **eliminar ou reduzir a probabilidade** *(likelihood)* de um ataque bem-sucedido

2. **Exemplos de Controles Preventivos**
	- As fontes [[0. FONTE]] listam **Controles Técnicos** que frequentemente atuam como preventivos:
		- **Listas de Controle de Acesso (ACLs):** Restringem o acesso antes que uma ação **NÃO AUTORIZADA ocorra.**
		- **Software Antivirus (AV) ou Endpoint Detection & Response (EDR):** Atuam antes que o malware consiga infectar ou executar uma ação maliciosa

3. **Diferença em Relação a Outros Tipos**
É crucial distinguir o controle preventivo dos outros tipos:

|            **Tipo de Controle**            |                                                        **Ação**                                                         |
| :----------------------------------------: | :---------------------------------------------------------------------------------------------------------------------: |
|               **Preventivo**               |                                  Opera **antes** do ataque, reduzindo a probabilidade                                   |
|             **[[Detectivo]]**              |                           Identifica e registra (logs) ataques, mas não é feito para prevenir                           |
|               **Corretivo**                | Usado **após** o ataque ter concluído, para reduzir seu efeito (ex: *patching* de vulnerabilidade explorada ou backups) |
| **[[Dissuasório (Desencoraja o ataque)]]** |                           Desencoraja o atacante de tentar o ataque, mas pode não preveni-lo                            |
