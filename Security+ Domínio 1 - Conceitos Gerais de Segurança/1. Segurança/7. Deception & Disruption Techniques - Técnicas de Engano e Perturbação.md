1. **Objetivo**
	O objetivo dessas técnicas é desviar a atenção de ativos sensíveis e obter inteligência sobre as ameaças

**Os principais conceitos são:**

|            **Técnica**            | **Definição e Propósito**                                                                                                                                                                                                                                                                                 |
| :-------------------------------: | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
|        **Active Defense**         | Engajamento defensivo com o adversário. O propósito é dificultar e tornar os ataques mais **"caros"** para os atacantes, exigindo mais tempo e recursos                                                                                                                                                   |
| **Decoy Assets -- (Ativos Isca)** | Ativos falsos implantados para atrair invasores (iscas). Não contém conteúdo e recursos sensíveis                                                                                                                                                                                                         |
|           **Honeypot**            | Um sistema configurado para atrair os atacantes. Permite registrar registros de logs para **análises estratégicas, táticas e ferramentas do atacante** serve como um **alerta precoce de possíveis ataques.** É crucial que seja isolado do resto do ambiente                                             |
|           **Honeynet**            | Uma rede de isca (*decoy network*) inteira. Consiste em múltiplos recursos (*servidores, estações de trabalho, etc.*) e é usada para monitoramento mais extensivo de ameaças avançadas                                                                                                                    |
|    **Honeyfile & Honeytoken**     | Dados falsos (*arquivos e tokens*) que parecem verdadeiros e convicentes, mas são inúteis para a organização. <br>Os tokens podem ser chaves da AWS ou de nuvem únicas e inúteis.<br>Os arquivos podem conter **"beacons"** que contatam um servidor quando abertos, permitindo rastrear o ator da ameaça |
|   **Estratégias de Disrupção**    | Envolvem o uso de falsas informações para confundir o atacante, como registros de **DNS falsos, diretórios de *web servers* de iscas, e retornos falsificados para *port scans***                                                                                                                         |

Essas técnicas são projetadas para reduzir o escopo das possíveis ameaças, limitando a superfície de ataque.
